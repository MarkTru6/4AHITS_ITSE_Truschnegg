## Übung (Webserver Directory Permissions)

Experimente unter Kali Linux mit dem /var/www (Apache Web-Server) Verzeichniss.

Entferne die other/read permission
```
sudo chmod o-r /var/www
```
- ist cd /var/www noch möglich? Warum?

  A: Ja, weil man dafür die execution permission braucht.
  
- Kann der Inhalt des Directories durch den User Kali jetzt noch angezeigt werden?

  A: Nein, weil man zum anzeigen des Inhalts das read recht braucht
  
- Kann von /var/www/ aus in html gewechselt werden?

  A: Ja, wenn auf beiden Verzeichnissen execution vorhanden ist.
  
Setze nun das other/r Flag wieder und entferne das other/x Flag.
```
sudo chmod o+r /var/www
sudo chmod o-x /var/www
```
Was sind die Auswirkungen? Experimentiere.

A: Inhalt anzeigen ist möglich aber man kann nicht in das directory wechseln oder eine datei darin öffnen

## Übung (Webserver Directory Permissions II)

Erstelle unter Kali Linux in /var/www ein Verzeichnis data und sorge dafür, dass dieses Verzeichnis nur User der Gruppe www-data verwenden können. Ein Lesezugriff auf den Directoryinhalt durch alle anderen User soll möglich sein jedoch kein Lesen der Dateiinhalte. User der Gruppe www-data können Datei-Inhalte anderer User ansehen aber nicht verändern.

- Erweiterung: User der Gruppe www-data sollen die Dateien anderer User der gleichen Gruppe im Directory data nicht löschen bzw. umbenennen können.
- Erweiterung: Alle Dateien/Verzeichnisse die im Ordner data erzeugt werden sollen automatisch der Gruppe www-data zugeordnet werden.
```
sudo mkdir /var/www/data
sudo chown root:www-data /var/www/data
sudo chmod 750 /var/www/data
chmod 755 /var/www/data
chmod 640 datei.txt
```
Erweiterung:
```
sudo chmod +t /var/www/data
```
